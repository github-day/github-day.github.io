Q2[10 points]

Find all artists born in the 1970s in England who have independently released music (without collaborators). List them in alphabetical order by artist name.

File: q2_solo_artist

Hints: The artist_count field in artist_credit denotes the number of artists involved in a release. England is an area name. Only consider artists who is a Person.

Your output should look like this:

ARTIST_NAME

Your first row should look like this:

Bashful

解析

select distinct(a1.name) 
from artist_credit as ac 
JOIN (select a.name 
      from artist as a 
      JOIN area on a.area=area.id and area.name='England' 
      JOIN artist_type on artist_type.name='Person' and artist_type.id=a.type 
      where a.begin_date_year>=1970 and a.begin_date_year<1980) 
      as a1 
on ac.name=a1.name and ac.artist_count=1 
order by a1.name;

分为两个步骤，第一步是查询所有出生于70年代并且来自England并且artist_type是Person的人员。第二步是根据这个人员名单表去artist_credit表中根据artist_count查询每个人的音乐发行的参与者

输出

Dusty Wagon

Q3[10points]

这一题我使用的方式有点复杂。嵌套了5层sql，应该是最烂的解法。

Find the ten latest releases by female artists in the German language. Order the results by release date from newest to oldest, and then by release name and artist name alphabetically.

File: q3_german_female_artists

Details: If there are releases with identical names and dates by the same artist, treat them as a single release and avoid duplicate entries. Format the release_date in the result as YYYY-MM-DD, without adding a leading zero if the month or day is less than 10.

Your output should look like this:

RELEASE_DATE|RELEASE_NAME|ARTIST_NAME

Your first row should look like this:

2020-9-25|The Songs of Johannes Brahms ~ 10|Sophie Rennert

解析

该题涉及至少四个relation（数据表），如下图所示。一个release对应一个artist_credit，而一个artist_credit对应多个artist_credit_name，所以artist_credit可以被消去，直接使用一个release对应多个artist_credit。

erDiagram release ||--|| release_info : includes release ||--|{ language : uses release ||--|| artist_credit : by artist_credit ||--|{ artist_credit_name : comprises artist_credit_name ||--|| artist : maps artist ||--|| gender : has release { id INTEGER name TEXT artist_credit INTEGER status INTEGER language INTEGER comment TEXT } release_info { release INTEGER area INTEGER date_year INTEGER date_month INTEGER date_day INTEGER } language { id INTEGER name TEXT } artist_credit { id INTEGER name TEXT artist_count INTEGER } artist_credit_name { artist_credit INTEGER position INTEGER artist INTEGER name TEXT } artist { id INTEGER name TEXT begin_date_year INTEGER begin_date_month INTEGER begin_date_day INTEGER end_date_year INTEGER end_date_month INTEGER end_date_day INTEGER type INTEGER area INTEGER gender INTEGER comment TEXT } gender { id INTEGER name TEXT description TEXT }

思路是先获取language=‘German'的release作为re0，re0 join release_info获取年月日字段得到re1，re1 join artist_credit_name 获取一个release对应的各个artist作为re3，re3 join gender得到满足所有artist为女性的表。 该表用根据关键字排序，DISTINCT关键字去重，输出前十行。最后用printf函数把年日月转换为格式化日期字符串。

感觉有点像合并树的节点。每次挑选两个节点进行合并，直到剩下最后一个节点

Q4[10points]

List the artist groups with the longest names in each area where the area name starts with the letter "Z". Organize the final output by sorting first by area name and then by artist name in alphabetical order. If multiple groups within the same area share the longest names, include them all.

File: q4_longest_name

Hints: Group is an artist_type. If multiple areas have different id, but the same name, consider them as the same area.

Your output should look like this:

AREA_NAME|ARTIST_NAME

One of the rows in the output looks like the following:

Zaandam|TheBeatVillain

解析

一开始用的是下面这种方法，问题是无法处理一个area中存在多个最大值的情况。

select n2.area_name,n2.artist_name from
(
select max(length(n1.name))
from (
    select artist.name,artist.area
    from 
    artist 
    join artist_type 
    on artist.type=artist_type.id and artist_type.name='Group'
    ) as n1 
join area 
on area.name LIKE 'Z%' and n1.area=area.id
group by area.name
) as n2

后面利用CTE减少重复查询。思路是先查询artist、artist_type和area建一个表n1，包含了artist_name、area_name、length(aritst_name)等，然后n1 join area再通过 group by 分组建立表n2，这样n2包含了每个地区的artist_name的最大值，然后n2 join n1再查询一次，匹配n1中artist_name长度等于n2中对应地区的长度最大值。
